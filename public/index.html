How the JSON model works

Your drawing is translated into an array of strokes, each with a tool, color, size and a list of [x,y] points. This is sent to the server as text (JSON) and stored in SQLite.

Implementation Docs (Wiki)

  1. HTML5 <canvas> receives pointer events.
  2. We track drawing state and push stroke objects as you draw.
  3. Each stroke is pure JSON: {tool, color, size, points: [[x,y], ...]}.
  4. The JSON is text-only and sent via fetch to Express.
  5. Canvas can be reconstructed by replaying strokes on load.

  • Bootstrap 5 for responsive layout & components.
  • Parallax hero header via background-attachment and scroll effect.
  • Undo/redo; live JSON preview; PNG export.

We implement 14+ routes including health, auth, CRUD, search, and share. Inputs are validated with zod. Centralised 404 and error handlers give robust responses with proper status codes.

Drawings are modelled as JSON, shown live on the page, POSTed to /api/drawings, and re-rendered by replaying strokes after GET.

We use SQLite via better-sqlite3 for simple, durable storage. On server start, tables are created if needed.

Passwords are hashed with bcrypt; sessions via express-session. No plain-text storage or transmission.
